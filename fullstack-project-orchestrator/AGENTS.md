# Reglas del Orquestador

## Principios

- Arquitectura antes que implementación.
- Contrato antes que backend.
- Backend antes que frontend complejo.
- Seguridad antes que producción.
- Observabilidad antes que escala.
- Documentación antes que entrega final.

## Umbrales

MVP pequeño:
- Arquitectura simplificada
- Backend modular básico
- Testing mínimo viable
- Deploy simple

Producto SaaS:
- Modular monolith
- ADR obligatorios
- Observabilidad base
- CI/CD obligatorio

Escala alta:
- Trade-offs explícitos
- Métricas desde día 1
- Separación clara de dominios
- Performance audit temprano
