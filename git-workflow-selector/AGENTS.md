# Reglas de ejecución (git-workflow-selector)

## Principios

- Simplicidad > complejidad innecesaria.
- El workflow debe adaptarse al tamaño del equipo.
- Siempre justificar la elección de estrategia.
- Priorizar claridad sobre rigidez excesiva.

## Defaults recomendados

- main protegida
- feature/<nombre>
- Conventional Commits
- PR obligatorio antes de merge
- CI mínimo: lint + test
- Versionado semántico
