# Git & GitHub (estándares base)

## Branching (default)
- main: estable
- develop (opcional): integración
- feature/<tema>
- fix/<tema>
- chore/<tema>

## Commits
- Conventional Commits: feat, fix, chore, docs, refactor, test, perf
- Mensajes cortos, imperativos, con scope cuando aporte valor: feat(api): add auth middleware

## PR
- Checklist: build OK, lint OK, tests OK, descripción clara, screenshots si aplica UI
- Revisión: al menos 1 aprobación (si hay equipo)

## Issues
- Template con: contexto, criterio de aceptación, riesgos, definición de terminado
