# project-architecture-planner

Skill para definir la arquitectura inicial de un proyecto fullstack con estándares profesionales.

## Qué produce
- Arquitectura y patrones recomendados.
- Árbol de carpetas (frontend/backend o monorepo).
- Contrato de API (alto nivel).
- Modelo de datos (alto nivel).
- Workflow Git/GitHub (ramas, commits, PRs, CI).
- Checklist de seguridad/calidad.
- Plan de ejecución por fases (MVP → v1).

## Cuándo usarla
- Inicio de proyecto.
- Re-arquitectura/refactor estructural.
- Definir estándares de un repositorio.
- Convertir una idea en un plan de implementación.

## Ejemplos de prompts
- "Quiero iniciar un e-commerce PERN. Definime arquitectura, carpetas y plan MVP."
- "Necesito organizar un monorepo React + Express + Postgres, con CI y convenciones."
- "Tengo un proyecto Node/Express caótico: proponé una arquitectura ordenada por capas."

## Output esperado
El output siempre incluye: decisiones, estructura, API y data model (alto nivel), workflow GitHub, checklist y plan por fases.
