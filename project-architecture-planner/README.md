# project-architecture-planner

Skill para definir arquitectura inicial fullstack con estándares profesionales, incorporando:
- Descubrimiento de contexto
- Trade-offs y ADR
- (Opcional) salida C4 Context

## Qué produce
- Arquitectura y patrones recomendados (con umbrales).
- Árbol de carpetas (monorepo o multi-repo).
- Contrato de API (alto nivel).
- Modelo de datos (alto nivel).
- Workflow Git/GitHub (ramas, commits, PRs, CI).
- Checklist de seguridad/calidad.
- Plan de ejecución por fases (MVP → v1).
- ADRs breves para decisiones significativas.
- C4 Context opcional (textual/Mermaid).

## Cuándo usarla
- Inicio de proyecto.
- Re-arquitectura/refactor estructural.
- Definir estándares del repositorio.
- Convertir una idea en plan ejecutable.

## Ejemplos de prompts
- "Quiero iniciar un e-commerce PERN. Definime arquitectura, carpetas, trade-offs y plan MVP."
- "Necesito organizar un monorepo React + Express + Postgres con CI y convenciones."
- "Tengo un proyecto Node/Express desordenado: proponé arquitectura por capas con ADRs."
