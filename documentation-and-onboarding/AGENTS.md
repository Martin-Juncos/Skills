# Reglas de ejecución (documentation-and-onboarding)

## Principios

- Claridad antes que cantidad.
- Documentar decisiones, no obviedades.
- Facilitar incorporación rápida.
- Mantener consistencia con el proyecto real.

## Defaults

- README estructurado por secciones claras.
- Carpeta /docs para ADR.
- CONTRIBUTING.md en raíz.
- ONBOARDING.md con pasos técnicos.
