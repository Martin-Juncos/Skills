# Reglas de ejecución (react-feature-scaffold)

## Principios

- Arquitectura por feature (no por tipo de archivo global).
- Separar UI de lógica.
- Separar llamadas API en servicios.
- No duplicar estado innecesariamente.
- Manejar loading y errores explícitamente.

## Defaults

- Estructura por feature.
- Hook personalizado para lógica.
- Servicio API separado.
- Tailwind si no se especifica sistema de estilos.
